## �� セキュリティ
| 項目 | 基本方針 | 評価観点（メリット・拡張性） |
| :--- | :--- | :--- |
| **認証方式** | JWT（Access短命）+ Refresh構成。外部連携・API公開時は OAuth 2.1（Auth Code + PKCE） へ拡張。 | 標準準拠 / 拡張性 |
| **認可** | RBAC（Role → Permission）＋所有者チェック。将来的に属性ベース（ABAC）をポリシー追加。 | 柔軟な権限管理 / ABAC移行性 |
| **通信** | TLS 1.3/1.2 強制（HTTPからHTTPSへのリダイレクト）。 | TLS強制 / 証明書管理 |
| **データ保護** | 機微情報の最小化。保存時は At-Rest暗号化、表示・ログ出力時は マスキング を徹底。 | 暗号化 / マスキング |
| **脆弱性対策** | CI/CDパイプラインでの 自動検査（SAST/DAST） および継続的なパッチ管理。 | 自動検査 / ライフサイクル管理 |
